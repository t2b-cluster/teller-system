2.1.3 API Gateway — Kong

ใช้ Kong API Gateway เป็น Single Entry Point สำหรับทุก Request จาก Frontend โดย Kong จัดการ:

  • Request Routing — กระจาย Traffic ไปยัง Service ที่ถูกต้องตาม Path
  • Rate Limiting — จำกัด Request 100/นาที สำหรับ Transfer, 200/นาที สำหรับ Query
  • CORS — อนุญาต Cross-Origin Request พร้อม x-idempotency-key Header
  • Request ID Injection — เพิ่ม X-Request-ID ทุก Request สำหรับ Distributed Tracing
  • Centralized Logging — Log ทุก Request/Response ผ่าน Kong
  • Admin GUI — จัดการ Routes/Plugins ผ่าน http://localhost:8002

2.1.4 Authentication — Auth Service + JWT Bearer Token

ระบบ Authentication แยกเป็น Auth Service (:3005) ดูแลเรื่อง Identity โดยเฉพาะ ใช้ JWT Bearer Token สำหรับ Stateless Authentication ทุก API Request ต้องแนบ Authorization: Bearer <token> ใน Header

🔐

Login Flow

Teller ส่ง username/password → Auth Service ตรวจสอบ bcrypt hash → ออก Access Token (JWT 30 นาที) + Refresh Token (7 วัน)

🔑

Token Validation

ทุก Business Service มี JwtAuthGuard เรียก Auth Service GET /auth/validate ตรวจ Bearer Token ก่อนดำเนินการ

🔄

Token Refresh + Rotation

Frontend auto-refresh ก่อนหมดอายุ 2 นาที พร้อม Refresh Token Rotation (revoke token เก่าทุกครั้ง)

🚪

Logout + Revoke

Logout จะ revoke Refresh Token ทันที ป้องกันการใช้ token ที่หลุดออกไป

2.5 Frontend Design — Security, Auth & Performance

🔐

Login Page + Auth Gate

หน้า Login แยกต่างหาก ถ้ายังไม่ login จะเข้าหน้าอื่นไม่ได้ ใช้ AuthContext (React Context) จัดการ state ทั้ง app

🚫

CSP Header + Security Headers

Nginx เพิ่ม X-Frame-Options, X-Content-Type-Options, X-XSS-Protection, Referrer-Policy และ CSP ที่เข้มงวด

หน้าจอหลัก 4 หน้า: Login → โอนเงิน (Transfer) → ประวัติธุรกรรม (Transaction History + Infinite Scroll) → ตรวจสอบยอดคงเหลือ (Balance Inquiry + Cache indicator)

4.4 Deployment Strategy — Blue-Green on OpenShift

เลือกใช้ Blue-Green Deployment ผ่าน OpenShift Route เพราะเป็นระบบธนาคารที่ต้องการ Zero Downtime อย่างเคร่งครัด โดยเฉพาะในช่วง Business Hours

Blue (Current)
← 👥 Traffic |
Green (New)
← Smoke Test
  1. Deploy เวอร์ชันใหม่เป็น Green โดยไม่รับ Traffic
  2. รัน Smoke Test + Health Check บน Green
  3. สลับ OpenShift Route ให้ชี้ไปที่ Green (ใช้เวลา < 1 วินาที)
  4. เก็บ Blue ไว้เป็น Rollback — หากมีปัญหาสลับกลับทันที
  5. หลังยืนยันสำเร็จ ลบ Blue เก่าออก
⚠️
Business Hours Deploy: การ Deploy ในช่วงเวลาทำการต้องมั่นใจว่า Teller ทุกคนไม่ได้รับผลกระทบ ดังนั้น maxUnavailable จะตั้งเป็น 0 เสมอ และมี Manual Approval ก่อน Production Deploy
