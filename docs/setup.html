2. Prerequisites

  • Google Cloud Account พร้อม Billing enabled
  • gcloud CLI ติดตั้งแล้ว (หรือใช้ Cloud Shell)
  • Domain name สำหรับ Observability (เช่น *.teller.tvdoing.com)
  • GitHub repository สำหรับ source code
  • Cloudflare account (สำหรับ DNS management)
💡 คำสั่งทั้งหมดด้านล่างรันได้ใน Google Cloud Shell โดยตรง ไม่ต้องติดตั้งอะไรเพิ่ม

3. Step 1 — GCP Project Setup

# ═══════════════════════════════════════════════════ # Step 1: สร้าง Project และเปิด APIs ที่จำเป็น # ═══════════════════════════════════════════════════ # ตั้งค่า Project export PROJECT_ID="t2b-cluster" export REGION="asia-southeast1" export ZONE="asia-southeast1-b" # สร้าง Project (ถ้ายังไม่มี) gcloud projects create $PROJECT_ID --name="Teller System T2B" # ตั้ง Project เป็น default gcloud config set project $PROJECT_ID gcloud config set compute/region $REGION gcloud config set compute/zone $ZONE # เปิด APIs ที่จำเป็น gcloud services enable \ container.googleapis.com \ sqladmin.googleapis.com \ artifactregistry.googleapis.com \ compute.googleapis.com \ cloudresourcemanager.googleapis.com \ iam.googleapis.com \ servicenetworking.googleapis.com echo "✅ Project setup complete"

4. Step 2 — GKE Autopilot Cluster

# ═══════════════════════════════════════════════════ # Step 2: สร้าง GKE Autopilot Cluster # ═══════════════════════════════════════════════════ export CLUSTER_NAME="t2b-gke" export NAMESPACE="tellersystem-uat" # สร้าง GKE Autopilot Cluster gcloud container clusters create-auto $CLUSTER_NAME \ --region=$REGION \ --project=$PROJECT_ID \ --network=default \ --subnetwork=default # รอ cluster พร้อม (~5-10 นาที) gcloud container clusters get-credentials $CLUSTER_NAME \ --region=$REGION \ --project=$PROJECT_ID # สร้าง Namespace kubectl create namespace $NAMESPACE # ตรวจสอบ kubectl get nodes echo "✅ GKE Autopilot cluster ready"
⚠️ GKE Autopilot มีข้อจำกัด: ห้ามใช้ privileged: true, hostPath อนุญาตเฉพาะ /var/log/, ไม่มี DaemonSet สำหรับ node-exporter/cAdvisor (ใช้ GKE built-in metrics แทน)

5. Step 3 — Cloud SQL (SQL Server + PostgreSQL)

# ═══════════════════════════════════════════════════ # Step 3A: Cloud SQL for SQL Server (teller_db) # ═══════════════════════════════════════════════════ # สร้าง Private IP range สำหรับ Cloud SQL gcloud compute addresses create google-managed-services-default \ --global \ --purpose=VPC_PEERING \ --prefix-length=16 \ --network=default gcloud services vpc-peerings connect \ --service=servicenetworking.googleapis.com \ --ranges=google-managed-services-default \ --network=default # สร้าง Cloud SQL for SQL Server gcloud sql instances create teller-sqlserver \ --database-version=SQLSERVER_2022_ENTERPRISE \ --tier=db-custom-2-8192 \ --region=$REGION \ --network=default \ --no-assign-ip \ --root-password="TellerPass@123" \ --storage-size=50GB \ --storage-auto-increase \ --availability-type=REGIONAL \ --backup-start-time=02:00 # สร้าง Database gcloud sql databases create teller_db \ --instance=teller-sqlserver # ดู Private IP gcloud sql instances describe teller-sqlserver \ --format="get(ipAddresses[0].ipAddress)" echo "✅ Cloud SQL for SQL Server ready"
# ═══════════════════════════════════════════════════ # Step 3B: Cloud SQL for PostgreSQL (Kong) # ═══════════════════════════════════════════════════ gcloud sql instances create kong-postgres \ --database-version=POSTGRES_15 \ --tier=db-f1-micro \ --region=$REGION \ --network=default \ --no-assign-ip \ --root-password="T2b@Kong2024!" \ --storage-size=10GB gcloud sql databases create kong \ --instance=kong-postgres gcloud sql users create kong \ --instance=kong-postgres \ --password="T2b@Kong2024!" # ดู Private IP gcloud sql instances describe kong-postgres \ --format="get(ipAddresses[0].ipAddress)" echo "✅ Cloud SQL for PostgreSQL (Kong) ready"
# ═══════════════════════════════════════════════════ # Step 3C: Initialize Database Schema # ═══════════════════════════════════════════════════ # ใช้ Cloud SQL Proxy หรือ connect จาก GKE pod # Option 1: จาก Jenkins VM SQLSERVER_IP=$(gcloud sql instances describe teller-sqlserver \ --format="get(ipAddresses[0].ipAddress)") # รัน init.sql ผ่าน sqlcmd sqlcmd -S $SQLSERVER_IP -U sa -P "TellerPass@123" \ -C -i infrastructure/db/init.sql echo "✅ Database schema initialized"

6. Step 4 — Artifact Registry

# ═══════════════════════════════════════════════════ # Step 4: สร้าง Artifact Registry สำหรับ Docker images # ═══════════════════════════════════════════════════ export AR_REPO="t2b" gcloud artifacts repositories create $AR_REPO \ --repository-format=docker \ --location=$REGION \ --description="Teller System container images" # Configure Docker auth gcloud auth configure-docker ${REGION}-docker.pkg.dev # ตรวจสอบ gcloud artifacts repositories list --location=$REGION echo "✅ Artifact Registry ready" echo "Registry URL: ${REGION}-docker.pkg.dev/${PROJECT_ID}/${AR_REPO}"

8. Step 6 — Jenkins VM Setup

# ═══════════════════════════════════════════════════ # Step 6A: สร้าง Jenkins VM # ═══════════════════════════════════════════════════ gcloud compute instances create t2b-vm01-jenkins \ --zone=$ZONE \ --machine-type=e2-standard-4 \ --image-family=ubuntu-2204-lts \ --image-project=ubuntu-os-cloud \ --boot-disk-size=100GB \ --boot-disk-type=pd-balanced \ --tags=jenkins-server \ --scopes=cloud-platform \ --metadata=startup-script='#!/bin/bash apt-get update apt-get install -y docker.io systemctl enable docker systemctl start docker usermod -aG docker $USER ' # Firewall rule สำหรับ Jenkins UI gcloud compute firewall-rules create allow-jenkins \ --allow=tcp:8080,tcp:22 \ --target-tags=jenkins-server \ --source-ranges=0.0.0.0/0 echo "✅ Jenkins VM created"
# ═══════════════════════════════════════════════════ # Step 6B: ติดตั้ง Tools บน Jenkins VM # (SSH เข้า VM แล้วรัน) # ═══════════════════════════════════════════════════ # SSH เข้า VM gcloud compute ssh t2b-vm01-jenkins --zone=$ZONE # --- รันคำสั่งด้านล่างใน VM --- # Install Java (Jenkins requirement) sudo apt-get update sudo apt-get install -y openjdk-17-jdk # Install Jenkins curl -fsSL https://pkg.jenkins.io/debian-stable/jenkins.io-2023.key | \ sudo tee /usr/share/keyrings/jenkins-keyring.asc > /dev/null echo "deb [signed-by=/usr/share/keyrings/jenkins-keyring.asc] \ https://pkg.jenkins.io/debian-stable binary/" | \ sudo tee /etc/apt/sources.list.d/jenkins.list > /dev/null sudo apt-get update sudo apt-get install -y jenkins sudo systemctl enable jenkins sudo systemctl start jenkins # Install kubectl curl -LO "https://dl.k8s.io/release/$(curl -L -s \ https://dl.k8s.io/release/stable.txt)/bin/linux/amd64/kubectl" sudo install -o root -g root -m 0755 kubectl /usr/local/bin/kubectl # Install gcloud CLI (ถ้ายังไม่มี) curl https://sdk.cloud.google.com | bash exec -l $SHELL gcloud init # Install Node.js 20 curl -fsSL https://deb.nodesource.com/setup_20.x | sudo -E bash - sudo apt-get install -y nodejs # Install Docker sudo apt-get install -y docker.io sudo usermod -aG docker jenkins sudo systemctl restart jenkins # Install Trivy sudo apt-get install -y wget apt-transport-https gnupg lsb-release wget -qO - https://aquasecurity.github.io/trivy-repo/deb/public.key | \ gpg --dearmor | sudo tee /usr/share/keyrings/trivy.gpg > /dev/null echo "deb [signed-by=/usr/share/keyrings/trivy.gpg] \ https://aquasecurity.github.io/trivy-repo/deb $(lsb_release -sc) main" | \ sudo tee -a /etc/apt/sources.list.d/trivy.list sudo apt-get update sudo apt-get install -y trivy # Install Gitleaks wget https://github.com/gitleaks/gitleaks/releases/download/v8.18.2/gitleaks_8.18.2_linux_x64.tar.gz tar -xzf gitleaks_8.18.2_linux_x64.tar.gz sudo mv gitleaks /usr/local/bin/ # Connect to GKE gcloud container clusters get-credentials t2b-gke \ --region=asia-southeast1 \ --project=t2b-cluster \ --internal-ip echo "✅ Jenkins VM tools installed"

9. Step 7 — Deploy Application to GKE

# ═══════════════════════════════════════════════════ # Step 7A: Build & Push Docker Images # (รันจาก Jenkins VM หรือ local ที่มี Docker) # ═══════════════════════════════════════════════════ export AR_REGISTRY="asia-southeast1-docker.pkg.dev/t2b-cluster/t2b" # Authenticate Docker กับ Artifact Registry gcloud auth configure-docker asia-southeast1-docker.pkg.dev # Build & Push ทุก service SERVICES="auth-service transfer-service transaction-service reconciliation-service notification-service frontend" TAG="latest" for svc in $SERVICES; do echo "Building $svc..." docker build -t ${AR_REGISTRY}/${svc}:${TAG} -f services/${svc}/Dockerfile . docker push ${AR_REGISTRY}/${svc}:${TAG} echo "✅ $svc pushed" done echo "✅ All images pushed to Artifact Registry"
# ═══════════════════════════════════════════════════ # Step 7B: Deploy K8s Manifests # ═══════════════════════════════════════════════════ export NAMESPACE="tellersystem-uat" # Connect to GKE gcloud container clusters get-credentials t2b-gke \ --region=asia-southeast1 \ --project=t2b-cluster # Create namespace kubectl create namespace $NAMESPACE --dry-run=client -o yaml | kubectl apply -f - # Replace image tag placeholder sed -i "s|IMAGE_TAG_PLACEHOLDER|latest|g" k8s/*.yaml # Delete old jobs (if exist) kubectl delete job kong-setup-routes -n $NAMESPACE --ignore-not-found=true kubectl delete job kong-migration -n $NAMESPACE --ignore-not-found=true # Apply all K8s manifests kubectl apply -f k8s/ -n $NAMESPACE # Wait for rollout for svc in auth-service transfer-service transaction-service \ reconciliation-service notification-service frontend kong redis; do echo "Waiting for $svc..." kubectl rollout status deployment/$svc -n $NAMESPACE --timeout=300s || true done echo "✅ Application deployed to GKE"
# ═══════════════════════════════════════════════════ # Step 7C: Verify Application # ═══════════════════════════════════════════════════ # ดู pods ทั้งหมด kubectl get pods -n $NAMESPACE # ดู services kubectl get svc -n $NAMESPACE # ดู Kong external IP kubectl get svc kong -n $NAMESPACE \ -o jsonpath='{.status.loadBalancer.ingress[0].ip}' echo "" # ดู Frontend external IP kubectl get svc frontend -n $NAMESPACE \ -o jsonpath='{.status.loadBalancer.ingress[0].ip}' echo "" echo "✅ Application verified"

10. Step 8 — Deploy Observability Stack

# ═══════════════════════════════════════════════════ # Step 8A: Deploy Grafana Dashboards as ConfigMap # ═══════════════════════════════════════════════════ kubectl create configmap grafana-dashboards \ --from-file=observability/grafana/dashboards/ \ -n $NAMESPACE \ --dry-run=client -o yaml | kubectl apply -f - echo "✅ Grafana dashboards ConfigMap created"
# ═══════════════════════════════════════════════════ # Step 8B: Deploy Observability K8s Manifests # ═══════════════════════════════════════════════════ # Deploy observability stack kubectl apply -f k8s/observability.yaml -n $NAMESPACE kubectl apply -f k8s/observability-ingress.yaml -n $NAMESPACE # Wait for rollout for svc in prometheus grafana elasticsearch kibana jaeger \ otel-collector logstash redis-exporter; do echo "Waiting for $svc..." kubectl rollout status deployment/$svc -n $NAMESPACE --timeout=300s || true done # Check Filebeat DaemonSet kubectl get daemonset filebeat -n $NAMESPACE echo "✅ Observability stack deployed"
# ═══════════════════════════════════════════════════ # Step 8C: Verify Observability # ═══════════════════════════════════════════════════ # Check Ingress kubectl get ingress observability-ingress -n $NAMESPACE # Check Prometheus targets PROM_POD=$(kubectl get pod -n $NAMESPACE -l app=prometheus \ -o jsonpath='{.items[0].metadata.name}') kubectl exec $PROM_POD -n $NAMESPACE -- \ wget -qO- "http://localhost:9090/api/v1/targets" | \ python3 -c "import sys,json; data=json.load(sys.stdin); \ [print(f\"{t['labels']['job']}: {t['health']}\") \ for t in data['data']['activeTargets']]" # Check Grafana health kubectl exec -n $NAMESPACE deploy/grafana -- \ wget -qO- "http://localhost:3000/api/health" # Check Elasticsearch kubectl exec -n $NAMESPACE deploy/elasticsearch -- \ curl -sf "http://localhost:9200/_cluster/health" echo "" echo "═══════════════════════════════════════════════" echo " Observability URLs:" echo "═══════════════════════════════════════════════" echo " Grafana: http://grafana.teller.tvdoing.com" echo " Jaeger: http://jaeger.teller.tvdoing.com" echo " Kibana: http://kibana.teller.tvdoing.com" echo " Prometheus: http://prometheus.teller.tvdoing.com" echo "═══════════════════════════════════════════════"

13. Cleanup Script

⚠️ คำสั่งด้านล่างจะลบ resources ทั้งหมด — ใช้เมื่อต้องการ teardown environment เท่านั้น
# ═══════════════════════════════════════════════════ # CLEANUP: ลบ resources ทั้งหมด # ═══════════════════════════════════════════════════ export PROJECT_ID="t2b-cluster" export REGION="asia-southeast1" export ZONE="asia-southeast1-b" # 1. ลบ GKE Cluster gcloud container clusters delete t2b-gke \ --region=$REGION --project=$PROJECT_ID --quiet # 2. ลบ Cloud SQL instances gcloud sql instances delete teller-sqlserver \ --project=$PROJECT_ID --quiet gcloud sql instances delete kong-postgres \ --project=$PROJECT_ID --quiet # 3. ลบ Artifact Registry gcloud artifacts repositories delete t2b \ --location=$REGION --project=$PROJECT_ID --quiet # 4. ลบ Static IP gcloud compute addresses delete observability-ip \ --global --project=$PROJECT_ID --quiet # 5. ลบ Jenkins VM gcloud compute instances delete t2b-vm01-jenkins \ --zone=$ZONE --project=$PROJECT_ID --quiet # 6. ลบ Firewall rules gcloud compute firewall-rules delete allow-jenkins \ --project=$PROJECT_ID --quiet echo "✅ All resources cleaned up" echo "⚠️ อย่าลืมลบ DNS records ที่ Cloudflare ด้วย"
